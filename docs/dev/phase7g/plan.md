# Phase 7g. Колбаска = сделки (была торговля или нет), а не запись

Расширение phase 7 (админ-фронт, Гант покрытия): разворот семантики «колбаски». Сейчас полоска
отражает **факт записи** (интервал `[started_at .. now]`), а должна отражать **торговую активность**
(была торговля в бакете или нет + разрывы между сделками). Классификация разрывов и честная подложка по
связи — в **phase 7h**. Дизайн Stage 1 — в [../apply.md](../apply.md); реализация — в [apply.md](apply.md);
статус — в [report.md](report.md).

**Статус:** `PLANNED`. **Stage:** 1. **Зависимости:** phase 7b (`sessions$`/`window$`,
`makeProjector`/`makeInverseProjector`), phase 7f (посессионная проекция, `displayTz`, тултипы/
crosshair), phase 6a (`coverage_segment`, `CoverageTracker`). Готовит почву под phase 7c (дат-точные
сессии — корректный порог разрыва внутри торгового окна).

## Мотивация (проблема)

Колбаска рисуется из одного `coverage_segment` как сплошной интервал `[started_at .. ended_at|now]`,
причём правый край живой записи тянется к `now` **клиентским таймером** (`useNow(1000)` → CSS
`--now-pct`), не глядя ни на сделки, ни на связь. Отсюда три дефекта:

1. **Полоска врёт про поток.** «Есть полоска» = «стояло на запись», а не «шла торговля». Пустой
   инструмент (0 сделок) рисует такую же колбаску, как активно торгуемый.
2. **Обрыв связи невидим.** Если коннектор отвалился, сегмент остаётся `ended_at IS NULL`, и колбаска
   продолжает ползти к `now`, рисуя «я пишу», хотя данных нет.
3. **Разрывы не там.** Гант нужен, чтобы видеть **разрывы между сделками** (в торговый день пауза
   > ~1 мин = аномалия/потеря данных), а не разрывы между записями. Сейчас разрывы (`gaps`) считаются
   из `md_trade`, но рисуются как вычет из сплошняка — вывернуто наизнанку.

`tradeCount` уже приходит на фронт, но нигде не влияет ни на геометрию, ни на цвет.

## Ключевые концепции

- **Два слоя дорожки:**
  1. **Подложка (намерение/покрытие записи)** — из `coverage_segment`. В 7g остаётся как есть
     (интервал `[started_at .. now]`); **честная подложка по состоянию связи — в phase 7h** (рвётся на
     обрыве/ошибке). 7g не полагается на её честность — только рисует поверх слой сделок.
  2. **Реальные сделки (основной слой)** — из `md_trade`, сведённые в бакеты. Отвечает на вопрос
     «данные *реально* приходили».
- **Присутствие, а не объём (качественно, не количественно).** Важно одно: **была торговля в бакете
  или нет**. Бакет со сделками → сплошная ячейка цветом источника; пустой → разрыв. Число сделок/объём
  **не визуализируем** — это НЕ heatmap и НЕ гистограмма объёмов как в терминалах.
- **Разрыв = отсутствие ячеек сделок.** Классификация разрыва (тихий рынок vs обрыв связи) — задача
  7h (через честную подложку + красная разметка обрывов). В 7g разрыв показываем нейтрально.
- **Статический бакет по таймфрейму (не по пикселям).** Бакет — фиксированный временной шаг на каждый
  таймфрейм (лесенка ниже), а не «px в время». Это делает агрегаты **кэшируемыми** (стабильный ключ) и
  предсказуемыми. Экран просто рисует эти бакеты.
- **Живой край — по данным, не по часам.** Правый край слоя сделок = время последней сделки; подложка
  тянется к `now`. `now`-таймер остаётся только для подложки и now-линии.

## Целевой рендер (схема)

```
          16:00         16:09              16:20 (now)
подложка  ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒   ← coverage_segment, до now (честность → 7h)
сделки    ██████ █ ███                            ← бакеты со сделками (сплошные), до последней (16:09)
                        └──── только фон ─────┘    ← «данных нет» (классификация → 7h)
```

Пример «редкий поток» (3 сделки за час, окно 1 день, бакет 1 мин):
```
16:00        17:00
▒▒▒█▒▒▒▒▒█▒▒▒▒▒▒▒▒▒▒█▒▒   ← подложка ровная, 3 сплошных бакета-тика (есть сделки) на фоне
```

## Лесенка бакетов (статическая, по таймфрейму)

Бакет фиксирован на каждый таймфрейм (монотонно растёт с горизонтом). Стартовые значения (уточняемо):

| Таймфрейм / горизонт | Бакет |
|---|---|
| D1 (1 день) | 30 c |
| D2–D3 | 1 мин |
| W (неделя) | 5 мин |
| M (месяц) | 30 мин |
| Q (квартал) | 2 ч |
| Y (год) | 12 ч |
| All | 1 день |

Кастомный диапазон → ближайшая ступень по числу дней. Не зависит от ширины экрана (кэш стабилен).

## Модель агрегации (бинарное присутствие) + кэш

- **Присутствие через `time_bucket`** (только непустые бакеты, без объёма):
  ```sql
  SELECT time_bucket(@bucket, ts) AS b
  FROM md_trade
  WHERE instrument_id = @id AND source_id = @src AND ts >= @from AND ts < @to
  GROUP BY b ORDER BY b;
  ```
  По индексу `ix_md_trade_instrument_ts` дёшево.
- **Кэш — прикладная таблица (Option A, ленивое наполнение).** `trade_activity_bucket(instrument_id,
  source_id, bucket_size, bucket_ts)` — храним только **непустые** бакеты. На запрос:
  1. читаем кэшированные бакеты окна;
  2. досчитываем недостающие **только для закрытых** бакетов (`< now`) и upsert-им;
  3. **открытый** бакет (где `now`) и хвост считаем на лету, не кэшируем (ещё не финальный).
  `md_trade` append-only → закрытые бакеты неизменны. Инвалидация — только при backfill (7c/9): чистим
  бакеты затронутого диапазона.
  > caggs (continuous aggregates) не берём: они материализуют **все** инструменты жадно; прикладной
  > кэш наполняется лениво — только тем, что реально смотрят. caggs — запас на будущий масштаб.
- **Батч-запрос** для всех видимых инструментов сразу — эндпоинт принимает список `instrumentId` +
  `bucket` + окно (почти всё cache-hit + маленький досчёт хвоста).

## Вне области (out of scope → позже)

- **Честная подложка по состоянию связи + классификация разрывов (тихо/обрыв) + красная разметка** —
  это **phase 7h** (автомат связи, `server_status` + пинг, эмуляция). 7g рисует разрыв нейтрально.
- **Continuous aggregates / materialized rollups** — под большие объёмы (phase 8+/масштаб).
- **Дат-точные торговые часы** — берём из phase 7c (ISS).
- **Backfill/эталонная сверка** (были ли сделки в непокрытом участке) — phase 7c (ISS) / 9 (qsh) /
  мультиисточник.
- **Мультиисточник в одной дорожке** (наложение цветов разных `source_id`) — сейчас фильтр по
  `connection.sourceId`.
- **Order-book/лента-превью** в тултипе — follow-up.

## Критерии приёмки

1. Колбаска рисуется из **сделок** (бинарно есть/нет): пустой инструмент (0 сделок) не показывает
   ячеек; торгуемый — сплошные ячейки в бакетах со сделками; редкий поток — отдельные тики на фоне.
2. **Подложка** отражает запись (до `now`), **отдельно** от слоя сделок (честность связи — 7h).
3. **Разрывы между сделками** видны как пустые бакеты (нейтрально; классификация — 7h).
4. **Бакет статичен по таймфрейму** (D1=30с … All=1д); не зависит от ширины экрана.
5. **Кэш работает:** повторный/пересекающийся запрос закрытых бакетов идёт из `trade_activity_bucket`,
   пересчитывается только хвост; UI отзывчив на дневном объёме и на многих инструментах (батч).
6. Живой край слоя сделок двигается **по последней сделке**, а не по настенным часам.
7. `tsc --noEmit` + `vitest` + backend-тесты зелёные.
