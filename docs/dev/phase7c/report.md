## Открытые пункты

- **Отложено за MVP (осознанно):** 7c.8 (лента новостей) и 7c.9 (двухслойный статус борда в карточке).
  Данные-фундамент (ISS-календарь, расписание сессий) уже есть; фичи вынесены — новости к разделу
  «Новости», статус борда — ближе к слою живости [7h](../phase7/roadmap.md).
- Источник расписания — публичный ISS (`https://iss.moex.com/iss`), без авторизации. Все ссылки и
  разбор таблиц — в [apply.md](apply.md).
- Наполнение `instrument`-каталога из ISS `securities` — не делаем: ISS используется для расписания,
  справочной «Структуры» и категоризации; запись котировок — через TRANSAQ/коннекторы.
- Горизонт достоверности календаря: часы — из недельного `timetable` (проекция на любую дату),
  праздники/переносы — из конечного `dailytable` (публикуется на текущий/следующий год); дальше дни
  ложатся по правилу недели.

## Итог

MVP фазы 7c закрыт: эвристика расписания заменена **реальными данными MOEX ISS** через бесплатный
`/iss/engines/{engine}` (`timetable`+`dailytable`). `/api/sessions` отдаёт дат-точные часы (регламентные/
сокращённые), праздники исключаются, ДСВД-выходные — по нашим данным с узким окном; при недоступности
ISS — прозрачный фолбэк на `MoexSchedule`. Календарь кэшируется в памяти (персистентность → Phase 13).
Добавлены read-only «Биржи → Структура» (движки/рынки/борды/инструменты, виртуализация, фильтры,
категоризация фьючерсов из ISS) и вкладка «Календарь» (месячные сетки с раскраской дней и часами).
Отложены за MVP (осознанно): лента новостей (7c.8 → раздел «Новости») и двухслойный статус борда в
карточке (7c.9 → ближе к слою живости 7h).
