# Phase 7h. Состояние связи + честная подложка + классификация разрывов

Расширение phase 7 (админ-фронт, Гант). Делает **подложку честной**: она отражает не «стояло на
запись», а **реальное покрытие захвата** (запись включена **и** связь жива). Тогда разрыв в потоке
сделок однозначно классифицируется: «тихий рынок» (подложка есть) vs «обрыв/не писали» (подложки нет).
Дизайн Stage 1 — в [../apply.md](../apply.md); реализация — в [apply.md](apply.md); статус — в
[report.md](report.md).

**Статус:** `PLANNED`. **Stage:** 1. **Зависимости:** phase 6a/6b (`coverage_segment`,
`ConnectionManager`, `ConnectorSession`), phase 7e (провайдеры/тумблер статуса), phase 7g (слой сделок
поверх подложки). Открывает дорогу к backfill (7c ISS / 9 qsh).

## Мотивация (проблема)

Подложка = `coverage_segment`, у которой правый край тянется к `now` **клиентским таймером**,
**не глядя на связь**. `server_status` от TRANSAQ обрабатывается **только на этапе connect**
(`TransaqConnector.TryCompleteConnect` выходит после установки) — обрыв **посреди сессии игнорируется**.
Результат: коннектор отвалился, а колбаска рисует «я пишу». Значит по подложке нельзя отличить
«тихо торговали» от «потеряли связь».

## Ключевая идея (дискриминатор)

**Есть подложка в момент дыры → мы захватывали → дыра = тихий рынок. Нет подложки → мы не захватывали
(обрыв/стоп/падение) → дыра = не покрыто (кандидат на backfill).**

Жёсткое правило: **подложка рвётся ТОЛЬКО по явному сигналу «связь вниз»/«стоп»/отсутствию живости,
НИКОГДА по тишине в сделках** (иначе тихий рынок ложно попадёт в «обрыв»).

## Модель: три независимых слоя дорожки

1. **Намерение** (`coverage_segment`) — юзер нажал запись `start..stop`. Правый край больше НЕ тянется
   к `now` слепо.
2. **Живость** (`capture_liveness`, новая сущность) — **положительное свидетельство жизни** в компактной
   **интервальной** форме `(source_id, from_ts, to_ts, open)`: пока связь жива, хартбит раз в `min_bucket/2`
   = **15 c** (мин. бакет лесенки 7g = 30 c) просто двигает `to_ts` открытого интервала. «Я живой, пока
   пишу». Пишем **на подключение (source)**. Компактно by design (рост ~ по числу обрывов, не по времени);
   **crash-safe**: `to_ts` = время последнего подтверждённого хартбита, при падении интервал замирает, при
   рестарте открывается новый → дыра честная автоматически. Сырого realtime-лога НЕ держим.
3. **Сделки** (`md_trade`, уже есть из 7g) — были ли реально сделки.

**Честная подложка = Намерение ∩ Живость.** Дыра в живости внутри намерения = настоящий обрыв. Дыра в
сделках при живой подложке = тихий рынок. Это положительная модель: при аварийном падении хоста хартбиты
не писались → в данных **реальная дыра без всякого graceful-close** (чинит «склейку» рестарта сама собой).

## Живость: правило сэмпла (надёжный паттерн)

На каждом тике живости (шаг 15 c) для активного подключения — по приоритету:

1. **`server_status = false/error`** в слоте → **down**: сэмпл `ok` НЕ пишем (→ дыра в подложке), драйвит
   закрытие сегмента с причиной.
2. **Пришла сделка** в слоте (или `now - lastData ≤ порог`) → **ok** без пинга. «Сделка = жив» —
   пинговать бессмысленно, и так понятно, что всё живо.
3. **Тишина**, но `server_status` ещё `connected` → **активный пинг** (дешёвый `SendCommand`): ответ ок →
   пишем `ok`; фейл/таймаут → **down**. Ловит «тихую смерть» DLL, не трогая тихий рынок.

То есть пинг будит только слоты неопределённости (тишина); поток сделок даёт `ok` бесплатно.
`server_status` слушаем **непрерывно** (авторитетный сигнал) — мгновенный `down` ловится независимо от
тайминга пинга.

## Конечный автомат связи

```
Disconnected → Connecting → Live ⇄ Degraded → Down/Error → (reconnect) → Live
```
- `connected="true"` → Live; `"false"`/`"error"` → Down/Error; `recover="true"` → Degraded.
- **Ре-подписка после реконнекта.** На `Live` после Down TRANSAQ не помнит подписки → `ConnectionManager`
  обязан заново подписать все активные записи (`RecordingManager`), иначе «connected, а сделок нет».
- **Пуш на фронт.** WS-событие `connectionStateChanged { connectionId, state, since, reason }` → тумблер
  провайдера + поведение подложки.

## Честная подложка: намерение × живость × причина

- **Геометрию** честности даёт живость (`capture_liveness`): подложка инструмента = его `coverage_segment`,
  **пересечённый с интервалами живости** его провайдера. Зазор между интервалами живости = дыра подложки.
- **Причину** разрыва (цвет) даёт статус закрытия `coverage_segment`: `error`/`disconnected` (красное) vs
  `stopped` (серое, сами остановили). Поэтому сегменты всё равно закрываем по состоянию связи — для «почему».
- **Recovery на старте хоста.** «Осиротевшие» открытые сегменты прошлого процесса (`ended_at IS NULL`)
  закрываем на запуске (статус `interrupted`, `ended_at` = время последней сделки/последнего хартбита),
  иначе устаревшее намерение будет ложно тянуться к `now`.
- **Переходы:** `Live` → сегмент открыт, растёт; Down/Error → закрываем (`disconnected`/`error`); Стоп →
  `stopped`; возврат в `Live` при активной записи → открываем новый сегмент.

## Разметка разрывов (цветом)

- **Тихий рынок** (подложка есть, ячеек сделок нет) — нейтрально.
- **Обрыв/ошибка** (сегмент закрыт `error`/`disconnected`) — **красная разметка**: у прямоугольника
  подложки `border-left: 1px solid red` в точке обрыва (и/или красная полоска-«шов»). `stopped` —
  нейтральный/серый (сами остановили).
- **Не писали** (нет сегмента, нет намерения) — пусто.

## Эмуляция разрыва (обязательно)

- `SyntheticLiveConnector` — инжект разрыва: `server_status connected="false"` → пауза → `recover` →
  `true`. Прогоняем весь путь: автомат → закрытие сегмента → WS → красная дырка → ре-подписка →
  новый сегмент.
- Дебаг-эндпоинт `POST /connections/{id}/debug/drop?seconds=N` — рукотворный разрыв на живом стенде.
- Реальный Transaq: выдернуть сеть/VPN, убедиться что ловим `server_status=false` и подложка рвётся.

## Область (in scope)

- **7h.0 host: recovery осиротевших сегментов на старте.** На запуске закрыть открытые сегменты
  (`ended_at IS NULL`) прошлого процесса: `status='interrupted'`, `ended_at` = время последней сделки
  сегмента (иначе `started_at`). Чинит «склейку» после аварийного рестарта. Миграция: добавить статус
  `interrupted` в `ck_coverage_status`. Полностью тестируемо (Testcontainers), без DLL.
- **7h.1 storage: живость (`capture_liveness`, интервалы).** ✅ DONE. Миграция `V010`
  `capture_liveness(liveness_id, source_id, from_ts, to_ts, open)` + уникальный индекс на один открытый
  интервал на source; `V011` добавляет `close_reason` (`stopped|server_down|ping_failed|interrupted`) +
  инвариант «open ⟺ reason NULL». `ICaptureLivenessStore`: `HeartbeatAsync` (открыть/продлить `to_ts`,
  split при пропуске тиков > maxGap = `interrupted`), `CloseAsync(sourceId, reason, atTs?)` (закрыть с
  причиной; `atTs` = точное время события, напр. `server_down`, сдвигает `to_ts`), `QueryAsync` (интервалы
  окна), **`QueryGapsAsync`** (журнал разрывов — производное: негативное пространство между интервалами,
  причина = `close_reason` предыдущего, `stopped` исключён), `RecoverOpenIntervalsAsync` (→ `interrupted`).
  Разрыв строкой НЕ хранится (положительная модель, crash-safe). Тесты: split/close-reason/server_down
  `to_ts`/журнал разрывов (исключение `stopped`, длящийся разрыв `To=null`) — зелёные.
- **7h.2 host: писатель хартбита + пинг.** Периодический тик **15 c = min_bucket/2** (Найквист: сэмпл
  вдвое чаще самого мелкого бакета 30 c → край разрыва локализуется с точностью до **одного** бакета;
  ловит кейс «в бакете были сделки, но захват рвался внутри» → подозрительный бакет для backfill). На тик:
  сделка/свежие данные → `HeartbeatAsync` (двигаем `to_ts`, при потоке сделок пинга НЕТ); тишина → активный
  пинг (`SendCommand`) → ok → `HeartbeatAsync`, fail → `CloseAsync` (обрыв). Реальный обрыв берёт
  `server_status` мгновенно — пинг лишь страхует «тихую смерть». Шаг в конфиг (`Ohs:LivenessProbeSeconds=15`).
  **Гейт пинга:** только при (1) активной записи, (2) торговых часах биржи (`MoexSchedule`), (3) тишине по
  сделкам. Вне сессии не пингуем (захватывать нечего; интервал живости закрывается к концу сессии) — это и
  снимает риск «пинговать всю ночь/выходные».
  На 7h.3 проверить, не шлёт ли TXmlConnector периодический `server_status` сам — тогда двигать `to_ts` по
  нему, активный пинг в тишине почти не нужен.
- **7h.3 connector: непрерывный `server_status`.** Вынести обработку из `TryCompleteConnect` в
  постоянный обработчик; коннектор публикует состояние связи (событие/`ConnectionState`), не только сигнал
  connect. `SyntheticLiveConnector` — уметь инжектить смену `server_status`.
- **7h.4 host: автомат + ре-подписка + причина.** `ConnectionManager` держит `ConnectorState`, реагирует
  на события связи; на реконнекте ре-подписывает активные записи; на Down/Error закрывает сегменты с
  причиной, на возврате в Live — открывает новые.
- **7h.5 WS + core.** Событие `connectionStateChanged { connectionId, state, reason }`; DTO/эндпоинт
  `/coverage/liveness`; `OhsStore.liveness$` + `refreshLiveness()` (батч по source), статус подключения.
  Правый край живой подложки двигать только пока state=Live.
- **7h.6 UI.** Подложка = `coverage_segment ∩ liveness` (с честными дырками); красная разметка обрывов
  (`error`/`disconnected`/`interrupted`), серое для `stopped`; тумблер провайдера учитывает Degraded/Down.
  Тултип: причина разрыва.
- **7h.7 эмуляция + тесты.** Быстрый визуальный тест — **ручной seed** `capture_liveness` (INSERT двух
  интервалов с зазором для source) → подложка рвётся честной красной дырой ещё до автоматики. Затем: инжект
  разрыва в synthetic + дебаг-эндпоинт `POST /connections/{id}/debug/drop`;
  тесты: recovery закрывает осиротевшие; обрыв рвёт живость и подложку; тихий рынок живость/подложку НЕ
  рвёт (пинг подтверждает ok); реконнект ре-подписывает; `connectionStateChanged` доходит.
  `tsc --noEmit` + `vitest` + backend-тесты зелёные.

## Критерии приёмки

1. Обрыв связи посреди сессии (`server_status=false`) **закрывает сегмент** и рвёт подложку; на фронте
   — дырка + **красная** разметка обрыва.
2. **Тихий рынок** (связь жива, сделок нет) подложку **не рвёт**; активный пинг подтверждает Live.
3. Реконнект **ре-подписывает** активные записи; после восстановления открывается новый сегмент и
   сделки снова текут.
4. `connectionStateChanged` пушится по WS; тумблер и подложка реагируют.
5. **Эмуляция разрыва** (synthetic-инжект / дебаг-эндпоинт) воспроизводит весь путь.
6. `tsc --noEmit` + `vitest` + backend-тесты зелёные.
