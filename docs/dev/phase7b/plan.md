# Phase 7b. Таймфреймы и сессионное окно (admin frontend)

Расширение phase 7 (админ-фронт): управление видимым окном Ганта через панель таймфреймов с
**сессионной** моделью MOEX. Дизайн Stage 1 — в [../apply.md](../apply.md); особенности реализации —
в [apply.md](apply.md); статус — в [report.md](report.md).

**Статус:** `PLANNED`. **Stage:** 1. **Зависимости:** phase 7 (Гант, `OhsStore`, `CoverageTrack`).

## Мотивация

Текущее окно Ганта — фиксированные «последние 2 часа», без управления. Нужен биржевой взгляд:
пользователь выбирает горизонт (сессия / неделя / месяц / квартал / год / всё / произвольные даты),
а ось времени статична и делится **по сессиям**. Минимальная единица — торговая сессия.

## Модель сессий (MOEX ЕТС, с 23.03.2026)

- Часовой пояс МСК = UTC+3 (без переходов на летнее время).
- Будний торговый день (ЕТС): `08:50–23:50` (утренняя доп. + основная + вечерняя, синхронизирован с
  календарным днём).
- Доп. сессия выходного дня (сб/вс, с 01.03.2025): `09:50–19:00` (аукцион открытия 09:50–10:00,
  основная 10:00–19:00). Не каждые выходные — есть список исключений (утверждает биржа); юридически
  это часть ближайшего рабочего дня (расчёты T+1). **Выходные не схлопываем** — показываем как
  отдельные слоты (пустые/подсвеченные); схлопывание станет отдельным фильтром.
- Правый край окна = конец **сегодняшней** сессии (в завтра не заглядываем); синяя now-линия —
  текущее время внутри сессии.

## Область (in scope)

- **7b.1 Backend: расписание сессий.** `MoexSchedule` (Domain) — часы сессии по типу дня
  (`Bounds(DateOnly) -> (start,end)`), запись `TradingSession { Date, Start, End, Weekend }`.
- **7b.2 Backend: торговые дни из данных.** `ICoverageStore.QueryTradingDaysAsync(count,
  includeWeekends)` — `DISTINCT` даты (МСК) из `md_trade`, фильтр выходных, `ORDER BY date DESC LIMIT`.
- **7b.3 Backend: экстент покрытия.** `QueryCoverageExtentAsync(sourceId?)` — `min(started_at)` /
  `max(coalesce(ended_at, now()))` из `coverage_segment` (для кнопки All).
- **7b.4 API.** `GET /api/sessions?count&includeWeekends` → `SessionDto[]`;
  `GET /api/coverage/extent[?sourceId]` → `CoverageExtentDto`. Контракт `IOhsApi` + клиент/фейки.
- **7b.5 core.** `Timeframe`-модель, `getSessions`/`getCoverageExtent`; `OhsStore`: `timeframe$`,
  `sessions$`, `includeWeekends$`, `setTimeframe(tf)`; замена авто-сдвига окна на сессионный пересчёт.
- **7b.6 UI: панель таймфреймов.** Дропдауны `D1–D30`/`W1–W4`/`M1–M12`/`Q1–Q4`/`Y1–Y10`, кнопка
  `All`, чекбокс «учёт выходных», диапазон дат from–to.
- **7b.7 UI: компоновка.** Строка `[TimeframePanel] [TimeAxis]`, пагинатор — отдельной строкой ниже
  (откат слияния из предыдущей итерации).
- **7b.8 UI: сепараторы сессий.** Вертикальные линии на границах сессий (`sessions$`) в
  `CoverageTrack`/`TimeAxis` — принцип «session by session — отделить».
- **7b.9 Тесты.** unit (`MoexSchedule`), integration (`QueryTradingDaysAsync`/extent), vitest
  (маппинг `Timeframe → window`).

## Маппинг таймфреймов → окно

- `D{n}` — последние n сессий.
- `W{n}` — последние `n×(5|7)` сессий (7 при включённом «учёте выходных», иначе 5).
- `M{n}`/`Q{n}`/`Y{n}` — календарный диапазон (n месяцев / n×3 месяцев / n лет) назад от сегодня,
  левый край привязан к началу ближайшей сессии, правый = конец сегодняшней.
- `All` — от `coverage/extent.from` до конца сегодняшней сессии.
- `Range(from,to)` — фиксированный диапазон (без live), привязан к границам сессий.

## Вне области (out of scope)

- Точный производственный календарь MOEX (праздники) как справочник — торговые дни берём из наличия
  данных; при отсутствии данных за день он просто не считается сессией.
- Пан/зум мышью по Ганту, drag-выделение диапазона — отдельный инкремент.
- Статус «торгуется/спит» (реальный `sec_status`) — по-прежнему отложен.

## Критерии приёмки

1. Панель таймфреймов переключает окно: `D1` — сегодняшняя сессия; `W1` — последняя неделя
   (5 или 7 сессий по чекбоксу); `All` — от самого раннего сегмента; выбор дат — фиксированный
   диапазон.
2. Ось времени статична и разделена по сессиям (видны границы), правый край = конец сегодняшней
   сессии, now-линия внутри.
3. `GET /api/sessions` возвращает торговые дни из данных с корректными часами (будни/выходные);
   `GET /api/coverage/extent` — границы покрытия.
4. `dotnet build` 0/0; unit/integration/api зелёные (добавлены новые); `tsc` + `vitest` зелёные.
