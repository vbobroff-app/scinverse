# Phase 7f. Тайм-лайн-фильтр оси Ганта + стандарт времени

Расширение phase 7 (админ-фронт): ось Ганта получает **тайм-лайн-фильтр** — управление тем, какие
дни показывать и какое окно внутри дня рисовать, плюс единый **стандарт времени** отображения на всю
систему. Дизайн Stage 1 — в [../apply.md](../apply.md); особенности реализации — в [apply.md](apply.md);
статус — в [report.md](report.md).

**Статус:** `IN PROGRESS (MVP)`. **Stage:** 1. **Зависимости:** phase 7b (`sessions$`/`window$`,
`makeProjector`, `TimeframePanel`), phase 7 (`OhsStore`, `InstrumentPicker`). Готовит почву под
phase 7c (реальные дат-точные расписания из ISS) и мультибиржу (Finam/Transaq).

## Мотивация

Ось Ганта строилась по единственной захардкоженной эвристике MOEX/МСК: все дни рисовались по одним
и тем же сессионным границам, время всегда в МСК. Это ломается на трёх сценариях:

1. **Внесессионные сделки не видны.** Пре-маркет/аукционы/вечёрка вне сессии «прилипают» к краю и
   теряются — а данные в базе есть.
2. **Разные площадки — разные сессии.** Через Finam→Transaq прилетают данные с зарубежных бирж
   (напр. S&P 500 с CME) со своим часовым поясом, расписанием и календарём. «Сессия» перестаёт быть
   свойством системы — это свойство **площадки**.
3. **Нет выбора часового пояса.** Аналитику может быть удобнее UTC или локальное биржевое время.

Нужен категоризированный, **расширяемый** фильтр оси и единый стандарт времени, отвязанный от
конкретного вида.

## Ключевые концепции

- **Сессия = атрибут площадки** (`exchange`/`board`), не глобальная константа. Определяется тройкой
  `{ tz, недельное расписание, календарь }`. Ключ уже есть в данных: `instrument.board → exchange`.
- **Данные не двигаются** (хранятся в UTC) — тайм-лайн-фильтр меняет только **проекцию оси** на
  клиенте: набор дней и границы каждого дня. Одинаково ко всем строкам (общая ось).
- **Общая ось — одна на все строки.** Поэтому `Сессия` (схлопывание внесессионного времени)
  когерентна только в разрезе одной площадки; `Полный день`/`Расписание` — кросс-биржевые и всегда
  корректны в миксе.
- **Стандарт времени — системная настройка отображения**, а не фильтр конкретного вида. Единый
  `displayTz` (смещение от UTC) применяется к оси, тултипам, подписям; сессии остаются привязаны к
  своим ТЗ — меняется только форматирование при выводе.

## Целевой UX

`[+]` в панели футера, в одном контейнере с `[Диапазон][D1][All]`. Поповер раскрывается вверх:

```
ДНИ
[Все][Будни][Сб, Вс] │ [🗓 MOEX][🗓 CME]
[Пн][Вт][Ср][Чт][Пт][Сб][Вс]
────────────────────────────
ОКНО ДНЯ
[Full] [🕐 moex] [🕐 cme]
[Расписание] [Smart]
  → Расписание: панель t1–t2 (time-picker)
  → Smart: [MOEX history][CME history][Set schedule]
                                        [Сбросить]
```

- **Дни** — набор дней недели (тумблеры Пн–Вс) + пресеты `Все/Будни/Сб, Вс` + торговые календари
  бирж (`🗓 MOEX`/`🗓 CME`, плейсхолдеры под 7c).
- **Окно дня** — `Full` (полные сутки) / сессия площадки (`🕐 moex`… по сегодняшнему расписанию,
  спроецированному на историю) / `Расписание` (окно t1–t2) / `Smart` (авто) + дат-точные расписания
  (`history`, плейсхолдеры).
- **`[+]`** подсвечивается синим, когда фильтр отклонён от нейтрали (все дни + `Full`).
- **`Сбросить`** возвращает нейтраль (все дни + `Full`). Стандарт времени — отдельно, в системных
  настройках (не сбрасывается фильтром).

**Стандарт времени** — вынесен в шапку: живые часы `HH:MM:SS МСК` + иконка темы + шестерёнка
настроек с `[UTC][МСК][UTC+N]` (последний настраивается). Меняешь стандарт → часы и вся ось
пересчитываются сразу.

## Область (MVP, in scope)

- **7f.1 core: модель.** `DayWindowMode` (`full`/`smart`/`session{exchange}`/`custom{fromMin,toMin}`),
  `TimelineFilter { weekdays, dayWindow }`, `DisplayTz { preset, offsetMin }` в `types.ts`.
- **7f.2 OhsStore.** `timelineFilter$`, `displayTz$`; `setTimelineFilter`/`resetTimelineFilter`/
  `setDisplayTz`; `shapeSessions` (фильтр дней недели + переразметка окна дня), `publishSessions`
  (применение + пересчёт `window$`), `resolveDayWindow`/`pickSmartExchange`; выходные для генерации
  выводятся из набора дней (`genIncludeWeekends`).
- **7f.3 Проекция.** Вся отрисовка уже идёт через `sessions$`+`window$` → трансформация сессий в
  сторе прокидывается в `TimeAxis`/`CoverageTrack` без их переделки.
- **7f.4 Стандарт времени.** `displayTz` прокинут в `TimeAxis`/`CoverageTrack` как `tzOffsetMin`;
  захардкоженный МСК убран, форматирование обобщено (`tzDateOf`).
- **7f.5 UI: `SessionFilter`.** Поповер `[+]` (дни/окно дня), внутри `TimeframePanel` (общий
  контейнер). Иконки — общий `icons.tsx` (`CalendarIcon`/`ClockIcon`, монохром, currentColor).
- **7f.6 UI: `HeaderControls`.** Живые часы + иконочный переключатель темы + системные настройки
  (шестерёнка) со стандартом времени; заменяет `ThemeToggle`.
- **7f.7 Тесты.** vitest на `OhsStore` (дни по фильтру, окно «полные сутки»); tsc/lint зелёные.

## Кастомные тултипы колбасок (7f.8, `DONE`)

Сейчас подсказка по колбаске/разрыву — нативный `title` (`CoverageTrack`): лаг появления (~0.5с,
не настраивается), системный вид, нельзя стилизовать, показывает только границы сегмента (не точку
под курсором). Заменяем на **кастомный тултип в отдельном слое**.

**Смысл.** На hover по колбаске всплывает тултип с **датой-временем точки под курсором** (а не только
границ сегмента) + контекст сегмента (источник, статус, число сделок, разрывы). Позиция курсора
переводится в момент времени **обратной проекцией** оси.

**In scope (7f.8):**

- **Обратная проекция.** `makeInverseProjector(fromMs, toMs, sessions)` в `sessionProjection.ts` —
  инверсия `makeProjector`: `pct (0..100) → ms`. Симметрично посессионной шкале (в шве — граница
  сессии). Покрыть vitest (round-trip `project(unproject(x)) ≈ x` на границах и внутри сессий).
- **Слой тултипа.** Единый оверлей поверх области дорожек (portal / фикс-слой), позиционируется по
  координатам курсора; НЕ ре-рендерит строки Ганта (мемоизация `CoverageTrack` сохраняется). Модель
  наведения — лёгкий стор/локальный стейт (`hoverTip$`: `{ x, y, ms, segment }`), обновляется на
  `pointermove` с throttle (rAF).
- **Данные точки.** Из наведённого сегмента: момент под курсором (обратная проекция + `tzOffsetMin`),
  источник (цвет+код), статус, сделок, разрывы. Уважает единый стандарт времени (`displayTz`).
- **Убрать `title`** с `.bar`/`.gap` (чтобы не дублировать нативной подсказкой).
- **Дизайн кастомного тултипа.** Тёмная карточка, скругл., тень, стрелка-указатель; заголовок —
  дата-время под курсором; строки контекста; цветовой маркер источника. Токены темы (свет/тёмная),
  без переполнения за вьюпорт (флип у краёв). Дизайн проработать здесь же.

**Вертикальный «прицел» (crosshair).** На наведение по **любой** дорожке (в т.ч. пустой без сделок)
системный курсор над дорожкой прячется, а его роль играет вертикальная линия по X курсора через все
дорожки; у оси снизу — подпись времени под курсором (`HH:mm:ss` в `displayTz`). Отдельный стор
`crosshair$` + оверлей `CrosshairOverlay` (геометрия из refs скролл-контейнера и ячейки оси).
Работает поверх той же обратной проекции; не ре-рендерит строки.

**Out of scope:** пиновка/клик-тултип, тултип-превью стакана/ленты — возможные follow-up.

## Вне области (out of scope, → phase 7c / позже)

- **Торговые календари бирж** (`🗓 MOEX`/`🗓 CME`) — реальные праздники/разовые торговые выходные из
  ISS. Сейчас плейсхолдеры (disabled).

- **Равные доли проекции** — сейчас доли пропорциональны длительности сессии (как в 7b); равные —
  возможный follow-up.
- **Персист фильтра/стандарта времени** в `user_settings` — при наличии phase 10.
- **`Полный день` для `All`** — режим All остаётся линейным (окно дня к нему не применяется).

## Критерии приёмки

1. `[+]` в панели футера открывает поповер; `Дни` фильтруют показанные дни, `Окно дня` меняет
   границы дня (Full = 00:00–24:00, session = сессия MOEX, Расписание = t1–t2).
2. Внесессионные сделки видны в режимах `Full`/`Расписание`; ось перестраивается сразу.
3. `[+]` подсвечен при отклонении от нейтрали; `Сбросить` возвращает все дни + `Full`.
4. Стандарт времени в шапке (UTC/МСК/UTC+N) меняет часы, подписи оси и тултипы синхронно.
5. По умолчанию — все дни + сессия MOEX (спроецированное текущее расписание).
6. (7f.8) hover по колбаске → кастомный тултип с датой-временем **точки под курсором** (обратная
   проекция) + контекст сегмента; нативный `title` убран; тултип не лагает и не ре-рендерит строки;
   уважает `displayTz`; корректно у краёв вьюпорта.
7. `tsc --noEmit` + `vitest` зелёные.
