# Фаза 3. Проверки — подробный план

**Цель.** Убедиться, что кодовая база и схема (после `V001`–`V003`) корректны: сборка без ошибок,
зелёные unit- и интеграционные тесты. Объём — **в необходимом объёме** для текущего состояния
(vertical slice write-path + миграции); новые тесты пишем только если обнаружится пробел.

## Исходное состояние (предусловия)

- Фазы 0–1 завершены: `V001`–`V003` накатаны, `Ohs.sln` собирается.
- Тестовые проекты уже существуют:
  - `Scinverse.Ohs.UnitTests` — `TickMath`, `TransaqXmlParser`, `TradeNormalizer` (+ `FakeInstrumentStore`);
  - `Scinverse.Ohs.IntegrationTests` — `TimescaleTradeWriter` на Testcontainers (`TimescaleFixture`).

## Задачи

| #   | Задача                                                    | Тип        |
| --- | -------------------------------------------------------- | ---------- |
| 3.1 | `dotnet build Ohs.sln` — сборка решения                  | проверка   |
| 3.2 | Прогнать unit-тесты                                       | проверка   |
| 3.3 | Прогнать интеграционные тесты (Testcontainers, нужен Docker) | проверка   |
| 3.4 | При зелёном прогоне — зафиксировать статус в [report.md](report.md) | правка |

## Критерии приёмки

1. Сборка `Ohs.sln` — 0 ошибок, 0 предупреждений.
2. Все unit-тесты — passed.
3. Все интеграционные тесты — passed (контейнер поднимается, миграции накатываются фикстурой,
   запись/чтение сделок и идемпотентность подтверждены).
4. Прогон воспроизводим локально одной командой (`dotnet test`).

## Вне объёма фазы

- Новые тесты под будущий функционал (деривативы, OrderLog, мультиисточник) — вместе с
  соответствующими фазами (1 расширенная / 2).
- Нагрузочное/перф-тестирование (COPY throughput) — отдельная активность при необходимости.
- CI-пайплайн (автоматический прогон на PR) — фаза деплоя.
