## Найденная регрессия и фикс (Npgsql 9)

Первый прогон интеграционных тестов упал (3 из 4): Npgsql 9 ужесточил бинарную запись —
`DateTimeOffset` с ненулевым смещением в `timestamptz` больше не принимается
(`Cannot write DateTimeOffset with Offset=03:00:00 ... only offset 0 (UTC) is supported`).
Регрессию внёс апгрейд Npgsql 8→9 из Фазы 0; Фаза 3 её и поймала.

**Фикс.** В `TimescaleTradeWriter.WriteAsync` момент времени приводится к UTC перед записью:
`trade.Timestamp.ToUniversalTime()` (смещение 0). Сохраняемое значение не меняется — `timestamptz`
хранит момент в UTC. После фикса — 4/4 зелёных.
