# Phase 4. Локальный E2E OHS (запись) — подробный план

**Цель.** Запустить OHS локально против живой TimescaleDB и убедиться, что write-path работает
end-to-end (данные лежат в `md_trade`). Плюс — потоковый демо-коннектор для «живых колбасок»
(непрерывный ингест во времени), нужный последующим фазам.

Часть Stage 1 (см. [../apply.md](../apply.md)). Верхнеуровневый статус — в [../plan.md](../plan.md).

## Исходное состояние (предусловия)

- Stage 0 завершён: `V001`–`V003` накатаны, TimescaleDB из `docker-compose` поднята.
- Хост `Scinverse.Ohs.Host` собран, DI/Serilog/опции настроены; конфиг: `UseFakeConnector: true`,
  строка подключения `localhost:5432`, инструмент `SBER/TQBR`.
- Write-path (parser → normalizer → batcher → `TimescaleTradeWriter` COPY) покрыт тестами.

## Задачи

| #   | Задача                                                                       | Тип        |
| --- | ---------------------------------------------------------------------------- | ---------- |
| 4.1 | Смоук: `dotnet run` хоста (fake) против живой БД; проверить `md_trade` + `instrument` | выполнение |
| 4.2 | Подключить **реальный TRANSAQ-коннектор** (исходник `TransaqConnector` уже есть): DLL, credentials, битность процесса; живой ингест в `md_trade` | выполнение |
| 4.3 | Отладка/наблюдаемость на реальном потоке: лог темпа ингеста; устойчивость при обрывах/переподключении | правка |

> Решение: от `SyntheticLiveConnector` отказались — есть доступный реальный TRANSAQ real-time
> коннектор, нет смысла тратить время на fake-стриминг.

## Критерии приёмки

1. Смоук-прогон наполняет `md_trade` (ожидаемо ~500 строк на инструмент для fake), `instrument`
   само-засеивается через `<securities>`.
2. Реальный TRANSAQ-коннектор подключается (Initialize + connect + subscribe), живые сделки пишутся
   в `md_trade` с корректными `ts`/ценой.
3. В логах виден темп/счётчик принятых сделок; коннектор устойчив к обрывам/переподключению.
4. `dotnet build` без ошибок; существующие тесты остаются зелёными.

## Предусловия для 4.2 (реальный TRANSAQ) — предоставляет оператор

- `txmlconnector.dll` нужной **битности** + путь (`Transaq:DllPath`). НЕ коммитим в репозиторий.
- Учётные данные: `Transaq:Login`/`Transaq:Password` через user-secrets или env (не в `appsettings.json`).
- Адрес шлюза: `Transaq:Host`/`Transaq:Port` (демо- или боевой сервер).
- **Битность процесса хоста обязана совпадать с DLL** (обычно txmlconnector 32-битный → нужен x86-хост).

- Control-plane (старт/стоп из UI), coverage, API/WS — phase6.
